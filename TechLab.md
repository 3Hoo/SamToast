# 이 연구를 시작하는 이유

1인 게임 개발에서 프로그래머가 크게 맞닥뜨리는 문제는 애셋일 것입니다.

2D 아트의 경우 여러 가지 로라의 조합과 필요한 경우 데이터셋을 구성하여 커스텀 lora를 훈련하여 원하는 바를 생성하기가 로컬 환경에서 달성하기 어렵지 않습니다.

문제는 3D AI의 경우, 지금까지 ai retopology를 포함하여 많은 발전이 있었으나, 버텍스의 노멀 벡터를 하나라도 줄이는 최적화가 중요시되는 게임에 있어서는 그대로 쓰기에 문제가 많은 것이 사실입니다. 허나 지금까지의 발전 경과를 보면, 분명 AI를 활용하면 3D 모델링, 리깅, 나아가 애니메이션의 프로토 타입 생성에 있어 효율성과 퀄리티를 높일 수 있을 것이라는 기대가 있습니다.

# 사용하고자 하는 기술

## 생성 환경

기본적으로 로컬 생성 환경은 ComfyUI로 구성하고, 이후 파이프라인이 완성되면 API화 하고자 합니다.

## 사용 모델/AI (로컬 위주로)

- 컨셉 이미지 생성 및 파츠별 분할
  - 로컬로는 DiffusionModel/Qwen/Flux 필요에 따라 사용할 것이나 현재로선 Flux2 Klein 기반 위주로 사용할 생각입니다
- 하이폴리 모델링
  - Trellis2/Hyunuan3D로 로컬 생성 -> Blender 스컵팅 기능으로 mesh cleanup & remesh & 디테일 보강
- 리토폴로지
  - MeshAnythingV2 : 정말 간단한 에셋의 경우
    - https://buaacyw.github.io/meshanything-v2/ 참고함
    - 가능하다면 comfyui custom node화
  - Blender QuadRemesher 플러그인
    - static 하고 애니메이션이 없어 관절 토폴로지에 신경을 쓰지 않아도 되는 경우
    - 유료 구매 필요
  - Blender Retopoflow
    - 수동 리토폴로지가 필요한 경우 사용. 4 버전을 직접 빌드 및 설치하여 사용
- UV 생성 & 노멀맵 베이킹
  - 이건 아직까진 AI 생성이 힘들어 보입니다. 따라서 Blender에서 직접 진행하고자 합니다
  - 가장 큰 병목이 될 수 있으므로, 기술 발전 양상의 지속적인 추적이 필요합니다
- 텍스처링
  - Blender StableGen
    - 사용할 ai 모델은 SDXL과 같은 diffustion 모델 또는 Flux 기반 모델
    - 후처리가 필요하다면 Blender Ucupaint 플러그인 또는 InstaMat 적절히 사용
- 리깅 & 스키닝
  - AccuRig (상용 서비스. 무료 티어로 사용할 예정)
  - ComfyUI-UniRig (UniRig 논문 기반의 comfyui 커스텀 노드)
  - 스키닝 후처리는 blender에서 직접 진행
- 애니메이션 (우선 휴먼 바디 대상)
  - HYMotion (Tencent)
  - Kimodo (Nvidia)
  - 기본적으로 UE 사용이 목적이므로 manny/quinn rig에 retarget 하여 ControlRig으로 프로시저럴 또는 직접 시퀀서에서의 수정으로 후처리

# 달성 목표 및 기대 성과

관련 지식 또는 최소한의 경험이 있는 인원들도 필요한 게임 (3D) 애셋을 효율적으로 생성가능하도록.

우선 기본적인 static한 에셋들에 대하여 ~텍스처링 단계까지 검증한 후, 캐릭터 애셋까지 하나의 정제된 워크플로우/파이프라인을 구축하는 것이 목적입니다.

이후에는 그 파이프라인을 skill, mcp, 또는 tool들로 정제하여서 llm agent 시스템(클로드 코드, openclaw 등)에서 다음과 같은 동작을 하는 것을 기대하고 있습니다

> 게임 개발 중 필요한 에셋 탐색 -> 새롭게 생성 필요한지 확인 -> 파이프라인 동작 시작 -> 각 동작 단계마다 휴먼 수정이 필요한 퀄리티인지 분별하여 human todo 작업에 추가
